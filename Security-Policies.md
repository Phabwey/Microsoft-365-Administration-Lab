# Security Policies

## Overview

Security policies are essential for protecting organizational data, managing access, and ensuring compliance within Microsoft 365 environments.

This lab demonstrates the implementation of security settings and administrative controls using Microsoft 365 and Microsoft Entra ID.

---

## Security Objectives

The following security objectives were implemented:

- Protect user accounts.
- Control access to organizational resources.
- Reduce security risks.
- Monitor user activity.
- Strengthen authentication methods.
- Improve data protection.

---

## Password Policies

The following password requirements were configured:

- Minimum password length: 8 characters
- Password complexity enabled
- Password expiration policy
- Password history enabled
- Self-service password reset

---

## Multi-Factor Authentication (MFA)

The following MFA policies were applied:

- MFA enabled for all users.
- Microsoft Authenticator configured.
- SMS verification enabled.
- Email verification enabled.

These policies improve account security and reduce unauthorized access.

---

## Role-Based Access Control (RBAC)

The following administrative roles were configured:

| Role | Responsibility |
|----------|----------|
| Global Administrator | Full administrative access |
| User Administrator | User management |
| Helpdesk Administrator | Password resets and support |
| Security Administrator | Security management |
| Teams Administrator | Teams management |
| Exchange Administrator | Email administration |

---

## Conditional Access Policies

The following access policies were implemented:

- Require MFA for administrative accounts.
- Block sign-ins from risky locations.
- Restrict access to sensitive resources.
- Enforce secure authentication methods.

---

## Security Monitoring

The following monitoring activities were performed:

- Reviewed user sign-in logs.
- Monitored account activity.
- Investigated suspicious logins.
- Audited administrative actions.

---

## Security Recommendations

The following recommendations were implemented:

- Enable Multi-Factor Authentication.
- Apply the principle of least privilege.
- Review permissions regularly.
- Monitor user activity.
- Keep systems updated.

---

## Administrative Tasks Performed

- Configured password policies.
- Enabled Multi-Factor Authentication.
- Assigned administrative roles.
- Applied Conditional Access policies.
- Reviewed security logs.
- Managed user permissions.
