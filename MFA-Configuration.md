# Multi-Factor Authentication (MFA) Configuration

## Overview

Multi-Factor Authentication (MFA) adds an additional layer of security to Microsoft 365 accounts by requiring users to verify their identity using multiple authentication methods.

This lab demonstrates the configuration and management of MFA within Microsoft 365 and Microsoft Entra ID.

---

## Authentication Methods

The following authentication methods were configured:

- Microsoft Authenticator application
- SMS verification
- Email verification
- Phone call verification

---

## MFA Configuration Process

The following steps were performed:

1. Opened the Microsoft Entra Admin Center.
2. Navigated to the user management section.
3. Selected the user account.
4. Enabled Multi-Factor Authentication.
5. Configured authentication methods.
6. Verified successful sign-in.

## Security Benefits

Multi-Factor Authentication improves security by:

- Preventing unauthorized access.
- Protecting user accounts.
- Reducing the risk of phishing attacks.
- Enhancing identity verification.
- Strengthening organizational security.

---

## Troubleshooting MFA Issues

The following issues were identified and resolved:

- Users unable to receive verification codes.
- Incorrect phone numbers.
- Authentication application synchronization issues.
- Lost or changed devices.
